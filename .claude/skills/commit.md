# commit

代码提交技能 - 使用中文编写规范的 Commit Message

## 使用方法

```bash
/skill commit
```

## 提交流程

1. **查看变更** - 运行 `git status` 和 `git diff` 查看待提交的更改
2. **分析改动** - 识别改动类型和功能/修复的内容
3. **选择类型** - 根据改动选择合适的类型
4. **编写信息** - 用中文编写规范的提交信息
5. **执行提交** - 添加文件并提交

## 提交类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新增功能 | feat: 添加用户登录功能 |
| `fix` | 修复 bug | fix: 修复表单验证失败问题 |
| `docs` | 文档更新 | docs: 更新 API 文档说明 |
| `style` | 代码格式调整 | style: 调整缩进和空格 |
| `refactor` | 重构代码 | refactor: 提取公共组件 |
| `perf` | 性能优化 | perf: 优化列表渲染性能 |
| `test` | 测试相关 | test: 添加单元测试用例 |
| `chore` | 构建/配置变更 | chore: 更新依赖版本 |
| `ci` | CI/CD 变更 | ci: 添加自动部署流程 |
| `revert` | 回滚提交 | revert: 回滚无效的更改 |

## 书写规则

### 标题行（必填）
- 格式：`<类型>: <描述>`
- 使用中文，祈使句风格
- 首字母无需大写，不以句号结尾
- 长度不超过 50 个字符

### 正文（可选）
- 详细描述改动内容和动机
- 复杂改动需说明原因和预期效果
- 每行不超过 72 个字符

### 脚注（可选）
- 关联 Issue：`Closes #123` 或 `Refs #456`
- 破坏性变更：`BREAKING CHANGE: 描述`

## 提交示例

### 新增功能
```
feat: 添加待办事项筛选功能

新增按状态筛选待办事项的功能
- 全部：显示所有待办
- 进行中：显示未完成的待办
- 已完成：显示已完成的待办

Closes #42
```

### Bug 修复
```
fix: 修复待办事项删除后状态异常

删除待办后未正确更新筛选状态，导致显示空白

Refs #58
```

### 代码重构
```
refactor: 重构 TodoStore 状态管理逻辑

提取通用的状态处理方法，减少代码重复

BREAKING CHANGE: useTodoStore 的 filter 方法签名变更
```

### 简单提交
```
docs: 更新 README 安装说明
```

```
style: 移除未使用的导入
```

## 注意事项

1. 提交前确保代码已通过 lint 检查
2. 确保相关测试已通过
3. 避免一次性提交过多不相关的改动
4. 提交信息应清晰表达"为什么改"而非仅"改了什么"
